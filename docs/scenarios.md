# 사용자 시나리오

## 시나리오 1. 과거 자기소개서 정리

1. 사용자가 과거 자기소개서 문항과 답변을 입력한다.
2. 시스템은 원문을 저장하고 경험 단위로 정리한다.
3. 사용자는 추출된 경험 목록을 확인한다.
4. 사용자는 각 경험의 출처와 부족한 정보를 확인한다.
5. 사용자는 보완 질문에 답변해 경험 정보를 더 완성한다.

## 시나리오 2. 포트폴리오 메모 정리

1. 사용자가 프로젝트 회고나 포트폴리오 메모를 입력한다.
2. 시스템은 프로젝트명, 역할, 행동, 성과, 기술 스택을 경험 카드로 정리한다.
3. 사용자는 성과 수치나 본인의 역할이 부족한 경험을 보완한다.

## 시나리오 3. 경험 검색

1. 사용자가 "백엔드 성능 개선 경험"처럼 찾고 싶은 경험을 검색한다.
2. 시스템은 사용자의 경험 chunk를 검색한다.
3. 사용자는 관련 경험과 원문 근거를 확인한다.

## 시나리오 4. JD/문항 기반 경험 추천

1. 사용자가 지원 직무, JD, 자기소개서 문항을 입력한다.
2. 시스템은 문항 의도와 요구 역량을 분석한다.
3. 경험 Vault에서 관련 chunk를 검색한다.
4. 검색된 chunk를 experience 단위로 묶는다.
5. LLM이 문항별 top-k 후보에 대해 추천/보류/비추천 경험을 판단한다.
6. 사용자는 추천 이유, 추천하지 않는 이유 또는 주의점, 원문 근거, 부족한 정보를 확인한다.
7. 사용자는 문항에 사용할 경험을 직접 선택한다.
8. 사용자는 선택한 경험의 부족한 정보에 답변하고, 시스템은 경험 Vault를 업데이트한다.

## 시나리오 5. 추천 경험 기반 자기소개서 초안 생성

1. 사용자가 지원 직무, JD, 자기소개서 문항, 글자 수 또는 톤을 입력한다.
2. 시스템은 문항 의도와 요구 역량을 분석한다.
3. 시스템은 경험 Vault에서 문항에 적합한 경험을 검색하고 문항별 top-k 후보를 추천한다.
4. 사용자는 추천 이유, 추천하지 않는 이유 또는 주의점, 원문 근거를 참고해 경험을 직접 선택하거나 시스템 추천 조합을 그대로 사용한다.
5. 부족한 정보가 있으면 사용자는 선택한 경험의 보완 질문에 답변한다.
6. 시스템은 선택 경험, 원문 근거, 보완 답변을 바탕으로 자기소개서 초안을 생성한다.
7. 사용자는 사이트에서 초안을 바로 수정한다.
8. 사용자는 문항별 초안을 txt 또는 md 파일로 export한다.
